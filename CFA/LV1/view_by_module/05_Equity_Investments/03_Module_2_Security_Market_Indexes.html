2.01 | Introduction

モジュール2：証券市場指数（Security Market Indexes） 2.01｜イントロダクション（Introduction）

投資家は、証券市場（security markets）に関する膨大な量の情報を継続的に収集し、分析している。この作業は時間がかかり、データ集約的であるため、投資家はしばしば、これらの情報を統合し、証券市場全体のパフォーマンスを反映する単一の指標（measure）を利用する。

証券市場指数（security market indexes）は、米国株式市場（US stock market）のパフォーマンスを反映するための単純な指標として最初に導入された。それ以来、証券市場指数は進化し、さまざまな証券市場のパフォーマンスの追跡、リスクの推定、投資マネージャーのパフォーマンス評価を支援する重要な多目的ツールとなっている。また、新たな投資商品（investment products）の基盤も形成している。

 

インデックス（index）, 名詞（複数形：indexes または indices） ラテン語の indic-（index）に由来し、indicare（指し示す）から派生：何かを示す指標（indicator）、兆候（sign）、または尺度（measure）。

市場指数（Market Indexes）の起源（Origin）

投資家は、1698年という早い時期からロンドン（London）において個別証券価格（individual security prices）に関する定期的に公表されるデータにアクセスできたが、証券市場の情報を反映する単純な指標にアクセスできるようになるまでには、ほぼ200年を要した。ある日の米国株式市場（US stock market）の全体的な動きを読者に伝えるために、出版者であるチャールズ・H・ダウ（Charles H. Dow）とエドワード・D・ジョーンズ（Edward D. Jones）は、1884年に世界初の証券市場指数であるダウ・ジョーンズ平均（Dow Jones Average）を導入した。この指数は『カスタマーズ・アフタヌーン・レター（The Customers’ Afternoon Letter）』に掲載され、9つの鉄道会社と2つの工業会社の株式で構成されていた。これは後にダウ・ジョーンズ輸送平均（Dow Jones Transportation Average）となった。将来の「主要な投機市場（the great speculative market）」は鉄道ではなく工業会社であると確信したダウとジョーンズは、1896年5月に第2の指数であるダウ・ジョーンズ工業平均（Dow Jones Industrial Average, DJIA）を導入した。この指数は初期値40.94で、米国の主要産業（major US industries）に属する12銘柄で構成されていた。現在では、投資家はさまざまな証券市場および資産クラス（asset classes）を測定・監視するために、数千の指数から選択することができる。

本リーディング（reading）の構成は以下のとおりである。第2節では証券市場指数（security market index）を定義し、単一期間および複数期間における指数の価格リターン（price return）とトータルリターン（total return）の計算方法を説明する。第3節では指数の構築および管理方法について説明する。第4節では市場指数（market indexes）の利用について論じる。第5節、第6節、および第7節ではさまざまな種類の指数について説明し、最後の節で本リーディングを要約する。結論および要約の後には練習問題（practice problems）が続く。

2.02 | Index Definition and Calculations of Value and Returns

モジュール2：証券市場指数（Security Market Indexes） 2.02｜指数の定義および値とリターンの計算（Index Definition and Calculations of Value and Returns）

Learning Outcome

The candidate should be able to:

  • 証券市場指数（security market index）を説明する
  • 指数の値（value）、価格リターン（price return）、およびトータルリターン（total return）を計算し、解釈する

証券市場指数（security market index）は、特定の証券市場（security market）、市場セグメント（market segment）、または資産クラス（asset class）を表す。ほとんどの指数は、市場で取引可能な証券（marketable securities）のポートフォリオとして構築される。

指数の値（value）は、指数に含まれる個々の証券（構成銘柄、constituent securities）の実際の市場価格または推定市場価格を用いて、定期的に計算される。各証券市場指数について、投資家は同一の指数（すなわち、同一の構成銘柄およびウェイトを持つ指数）に対して2つのバージョンに遭遇する可能性がある。すなわち、価格リターンに基づくバージョンとトータルリターンに基づくバージョンである。名称のとおり、価格リターン指数（price return index）、別名価格指数（price index）は、指数に含まれる構成銘柄の価格のみを反映する。一方、トータルリターン指数（total return index）は、構成銘柄の価格だけでなく、設定時（inception）以降に受け取ったすべての収益（income）の再投資も反映する。

設定時（inception）においては、価格リターン指数とトータルリターン指数の値は等しい。しかし、時間の経過とともに、配当（dividends）および／または利息（interest）のすべてを再投資するトータルリターン指数の値は、価格リターン指数の値をますます上回るようになる。複数期間にわたる各バージョンの値の計算方法を見ることで、その理由が理解できる。

価格リターン指数（price return index）の値は次のように計算される：

V P R I = ∑ i = 1 N n i P i D 1

where

  • V PRI ＝ 価格リターン指数の値（value of the price return index）
  • n i ＝ 指数ポートフォリオにおいて保有される構成銘柄 i の数量（number of units）
  • N ＝ 指数に含まれる構成銘柄の数（number of constituent securities）
  • P i ＝ 構成銘柄 i の単位価格（unit price）
  • D ＝ 除数（divisor）の値

除数（divisor）は、設定時（inception）に最初に選ばれる数値である。多くの場合、価格指数（price index）の初期値が1,000のような扱いやすい値になるように選定される。その後、指数提供者（index provider）は、構成銘柄の価格変動とは無関係な指数値の変化を避けるため、必要に応じて除数の値を調整する。例えば、構成銘柄を変更する際には、変更後の指数の値が変更前の指数の値と等しくなるように除数を調整することがある。

指数リターン（index return）の計算は、投資ポートフォリオ（investment portfolio）のリターン計算と同様に、価格リターンまたはトータルリターンを測定することができる。価格リターン（price return）は、価格上昇（price appreciation）または価格の変化率のみを測定する。トータルリターン（total return）は、価格上昇に加えて利息（interest）、配当（dividends）、およびその他の分配（distributions）を含む。

単一期間リターン（Single-Period Returns）の計算

証券市場指数（security market index）の価格リターンは、2つの方法で計算できる。すなわち、価格リターン指数の値の変化率、または構成銘柄の価格リターンの加重平均である。指数の価格リターンは次のように表される：

PR I = V P R I 1 − V P R I 0 V P R I 0 2

where

  • PR I ＝ 指数ポートフォリオの価格リターン（price return）（小数表示、例えば12％は0.12）
  • V PRI 1 ＝ 期間末における価格リターン指数の値
  • V PRI 0 ＝ 期間初における価格リターン指数の値

同様に、各構成銘柄の価格リターンは次のように表される：

PR i = P i 1 − P i 0 P i 0 3

where

  • PR i ＝ 構成銘柄 i の価格リターン（小数表示）
  • P i 1 ＝ 期間末における構成銘柄 i の価格
  • P i 0 ＝ 期間初における構成銘柄 i の価格

指数の価格リターンは個々の証券の価格リターンの加重平均に等しいため、次のように表すことができる：

PR I = ∑ i = 1 N w i PR i = ∑ i = 1 N w i ( P i 1 − P i 0 P i 0 ) 4

ここで：

  • PR I ＝ 指数ポートフォリオの価格リターン（小数表示）
  • PR i ＝ 構成銘柄 i の価格リターン（小数表示）
  • N ＝ 指数に含まれる個別証券の数
  • w i ＝ 証券 i のウェイト（指数ポートフォリオにおいて証券 i に配分される割合）
  • P i 1 ＝ 期間末における構成銘柄 i の価格
  • P i 0 ＝ 期間初における構成銘柄 i の価格

式（4）は次のように簡潔に書き直すことができる：

PRI = w1PR1 + w2PR2 + … + wN PRN  5

where

  • PR I ＝ 指数ポートフォリオの価格リターン（小数表示）
  • PR i ＝ 構成銘柄 i の価格リターン（小数表示）
  • w i ＝ 証券 i のウェイト（指数ポートフォリオにおける配分割合）
  • N ＝ 指数に含まれる証券数

トータルリターン（total return）は、価格上昇に加えて利息（interest）、配当（dividends）、およびその他の分配（distributions）を含む。したがって、指数のトータルリターンは、価格リターン指数の値の変化（価格上昇）に、期間中の収益（配当および／または利息）を加えたものであり、これを期間初の価格リターン指数の値に対する割合として表したものである。指数のトータルリターンは次のように表される：

TR I = V P R I 1 − V P R I 0 + I n c I V P R I 0 6

where

  • TR I ＝ 指数ポートフォリオのトータルリターン（小数表示）
  • V PRI 1 ＝ 期間末における価格リターン指数の値
  • V PRI 0 ＝ 期間初における価格リターン指数の値
  • Inc I ＝ 期間中に指数に含まれるすべての証券から得られる総収益（配当（dividends）および／または利息（interest））

指数のトータルリターン（total return）は、構成銘柄のトータルリターンの加重平均としても計算できる。指数に含まれる各構成銘柄のトータルリターンは次のように計算される：

TR i = P 1 i − P 0 i + I n c i P 0 i 7

where

  • TR i ＝ 構成銘柄 i のトータルリターン（小数表示）
  • P 1 i ＝ 期間末における構成銘柄 i の価格
  • P 0 i ＝ 期間初における構成銘柄 i の価格
  • Inc i ＝ 期間中に証券 i から得られる総収益（配当および／または利息）

指数のトータルリターンは構成銘柄のトータルリターンの加重平均として計算できるため、次のように表すことができる：

TR I = ∑ i = 1 N w i TR i = ∑ i = 1 N w i ( P 1 i − P 0 i + I n c i P 0 i ) 8

式（8）は次のように簡潔に書き直すことができる：

TRI = w1TR1 + w2TR2 + … + wN TRN  9

where

  • TR I ＝ 指数ポートフォリオのトータルリターン（小数表示）
  • TR i ＝ 構成銘柄 i のトータルリターン（小数表示）
  • w i ＝ 証券 i のウェイト（指数ポートフォリオにおいて証券 i に配分される割合）
  • N ＝ 指数に含まれる証券数

複数期間にわたる指数値（index values）の計算

複数期間にわたる指数値の計算には、指数リターン系列（series of index returns）を幾何的に連結（geometrically linking）する必要がある。指数の価格リターン系列を用いて、次の式により価格リターン指数の値を計算できる：

VPRIT = VPRI 0(1 + PRI 1)(1 + PRI 2)…(1 + PRIT)  10

where

  • V PRI 0 ＝ 設定時（inception）における価格リターン指数の値
  • V PRIT ＝ 時点 t における価格リターン指数の値
  • PR IT ＝ 期間 t における指数の価格リターン（小数表示）、t ＝ 1, 2, …, T

設定時の値が1,000であり、第1期および第2期の価格リターンがそれぞれ5％および3％である指数を考えると、価格リターン指数の値は次のように計算される：

期間（Period） リターン（Return）（％） 計算（Calculation） 期末値（Ending Value）
0   1,000（1.00） 1,000.00
1 5.00 1,000（1.05） 1,050.00
2 3.00 1,000（1.05）（1.03） 1,081.50

同様に、指数のトータルリターン系列（series of total returns）を用いて、次の式によりトータルリターン指数の値を計算する：

VTRIT = VTRI 0(1 + TRI 1)(1 + TRI 2)…(1 + TRIT)  11

where

  • V TRI 0 ＝ 設定時（inception）における指数の値
  • V TRIT ＝ 時点 t におけるトータルリターン指数の値
  • TR IT ＝ 期間 t における指数のトータルリターン（小数表示）、t ＝ 1, 2, …, T

同じ指数が、第1期に1.5％、第2期に2.0％の収益（income）による追加リターンを生むと仮定すると、第1期および第2期のトータルリターンはそれぞれ6.5％および5％となる。このとき、トータルリターン指数の値は次のように計算される：

期間（Period） リターン（Return）（％） 計算（Calculation） 期末値（Ending Value）
0   1,000（1.00） 1,000.00
1 6.50 1,000（1.065） 1,065.00
2 5.00 1,000（1.065）（1.05） 1,118.25

上記のとおり、時間の経過とともに、受け取ったすべての配当（dividends）および／または利息（interest）を再投資するトータルリターン指数（total return index）の値は、価格リターン指数（price return index）の値を、ますます大きな差で上回るようになる。

2.03 | Index Construction

モジュール2：証券市場指数（Security Market Indexes） 2.03｜指数の構築（Index Construction）

Learning Outcome

The candidate should be able to:

  • 指数の構築および管理における選択肢と課題を説明する
  • 指数構築に用いられる異なる加重方法（weighting methods）を比較する
  • 加重方法に基づいて指数の値およびリターンを計算し、分析する

証券市場指数（security market index）の構築および管理は、証券ポートフォリオの構築および管理に類似している。指数提供者（index providers）は、以下を決定しなければならない：

  1. 指数はどのターゲット市場（target market）を表すべきか？
  2. そのターゲット市場からどの証券を選択すべきか？
  3. 指数において各証券にどの程度のウェイトを割り当てるべきか？
  4. 指数はいつリバランス（rebalanced）されるべきか？
  5. 証券の選択およびウェイトの決定はいつ再検討されるべきか？

ターゲット市場（Target Market）と証券選択（Security Selection）

指数構築における最初の意思決定は、その指数が表すことを意図するターゲット市場（target market）、市場セグメント（market segment）、または資産クラス（asset class）を特定することである。ターゲット市場は、非常に広範にも狭義にも定義され得る。資産クラス（例：株式（equities）、債券（fixed income）、不動産（real estate）、商品（commodities）、ヘッジファンド（hedge funds））、地理的地域（例：日本（Japan）、南アフリカ（South Africa）、ラテンアメリカ（Latin America）、ヨーロッパ（Europe））、証券が取引される取引所（例：上海（Shanghai）、トロント（Toronto）、東京（Tokyo））、および／またはその他の特性（例：経済セクター（economic sector）、企業規模（company size）、投資スタイル（investment style）、デュレーション（duration）、信用の質（credit quality））に基づいて定義される場合がある。

ターゲット市場は、投資ユニバース（investment universe）および指数に組み入れ可能な証券を決定する。投資ユニバースが特定されると、指数に含める証券の数および具体的な銘柄を決定しなければならない。構成銘柄（constituent securities）は、ターゲット市場のほぼすべてである場合もあれば、その代表的サンプルである場合もある。S&P 500指数（S&P 500 Index）やFTSE 100（FTSE 100）のような一部の株式指数は、指数に含まれる証券数を固定し、その数を指数名に示している。他の指数は、ターゲット市場の変化を反映したり、ターゲット市場の一定割合を維持したりするために、証券数を変動させる。例えば、東証株価指数（Tokyo Stock Price Index, TOPIX）は、東京証券取引所（Tokyo Stock Exchange）に上場する最大規模の銘柄群である第一部（First Section）に属するすべての株式を対象としている。第一部、ひいてはTOPIXに組み入れられるためには、発行済株式数（number of shares outstanding）、株主数（number of shareholders）、時価総額（market capitalization）などの基準を満たす必要がある。これらの基準を満たさなくなった銘柄は第一部およびTOPIXから除外される。多くの指数では客観的または機械的なルールによって構成銘柄が決定されるが、すべてではない。例えば、S&Pボンベイ証券取引所センシティブ指数（S&P Bombay Stock Exchange Sensitive Index）、別名S&P BSE SENSEX、およびS&P 500は、選定委員会（selection committee）とより主観的な意思決定ルールを用いて構成銘柄を決定している。

指数の加重（Index Weighting）

加重の決定は、各証券を指数にどの程度組み入れるかを決定し、指数の値に大きな影響を与える。指数提供者は、構成銘柄にウェイトを割り当てるためにさまざまな方法を用いる。指数は、価格加重（price weighted）、等加重（equal weighted）、時価総額加重（market-capitalization weighted）、またはファンダメンタル加重（fundamentally weighted）とすることができる。それぞれの加重方法には利点と欠点がある。

価格加重（Price Weighting）

指数を加重する最も単純な方法であり、チャールズ・ダウ（Charles Dow）がダウ・ジョーンズ工業平均（Dow Jones Industrial Average）を構築する際に用いた方法が価格加重（price weighting）である。価格加重では、各構成銘柄のウェイトは、その価格をすべての構成銘柄の価格の合計で割ることによって決定される。ウェイトは次の式によって計算される：

w i P = P i ∑ i = 1 N P i 12

もう1つの単純な指数加重方法は等加重（equal weighting）である。この方法では、設定時（inception）に各構成銘柄に等しいウェイトを割り当てる。ウェイトは次のように計算される：

w i E = 1 N 13

where

  • w i ＝ 証券 i に配分されるポートフォリオの割合、すなわち証券 i のウェイト
  • N ＝ 指数に含まれる証券数

時価総額加重（Market-Capitalization Weighting）

時価総額加重（market-capitalization weighting）、またはバリュー加重（value weighting）では、各構成銘柄のウェイトは、その時価総額（market capitalization）を指数内のすべての証券の時価総額の合計で割ることによって決定される。時価総額（market capitalization）または価値（value）は、発行済株式数（number of shares outstanding）に1株当たりの市場価格（market price per share）を乗じることで計算される。

証券 i の時価総額ウェイトは次のとおりである：

w i M = Q i P i ∑ j = 1 N Q j P j 14

where

  • w i ＝ 証券 i に配分されるポートフォリオの割合、すなわち証券 i のウェイト
  • Q i ＝ 証券 i の発行済株式数（number of shares outstanding）
  • P i ＝ 証券 i の株価（share price）
  • N ＝ 指数に含まれる証券数

Float-Adjusted Market-Capitalization Weighting

フロート調整時価総額加重（float-adjusted market-capitalization weighting）では、各構成銘柄のウェイトは、その市場浮動株（market float）を考慮して時価総額を調整することによって決定される。通常、市場浮動株（market float）とは、投資家が売買可能な株式数を指す。株式の保有が集中している企業（closely held companies）では、発行済株式のうち一部のみが投資家に利用可能であり、残りは少数の支配的投資家（controlling investors）によって保有されている。支配株主が保有する株式を除外することに加え、多くのフロート調整時価総額加重指数では、他の企業や政府が保有する株式も除外する。また、グローバル投資家の投資機会を表すことを目的とした指数の一部の提供者は、外国人投資家（foreigner investors）が購入できない株式を除外することで、指数に含める株式数をさらに減らす。このような指数は「フリーフロート調整時価総額加重指数（free-float-adjusted market-capitalization-weighted indexes）」と呼ばれることがある。

フロート調整時価総額加重指数は、総発行株式数（total market capitalization）ではなく、投資家が利用可能な株式数（すなわちフロート調整時価総額、float-adjusted market capitalization）に株価を乗じることで、公に取引可能な株式を反映する。現在では、ほとんどの時価総額加重指数はフロート調整されている。したがって、本リーディングの残りの部分では、特に断りがない限り、「時価総額加重（market-capitalization weighting）」はフロート調整時価総額加重を指す。

証券 i のフロート調整時価総額ウェイトは次のように計算される：

w i M = f i Q i P i ∑ j = 1 N f j Q j P j 15

where

  • f i ＝ 発行済株式数のうち市場浮動株に該当する割合
  • w i ＝ 証券 i に配分されるポートフォリオの割合、すなわち証券 i のウェイト
  • Q i ＝ 証券 i の発行済株式数
  • P i ＝ 証券 i の株価
  • N ＝ 指数に含まれる証券数

時価総額加重（market-capitalization weighting）（フロート調整を含む）の主な利点は、構成銘柄がターゲット市場（target market）における価値に比例して保有される点である。主な欠点は、価格が最も上昇した（または下落した）証券が指数内でより大きな（または小さな）ウェイトを持つことである（すなわち、ある証券の価格が指数内の他の証券に対して相対的に上昇すると、そのウェイトは増加し、逆に相対的に下落するとウェイトは減少する）。この加重方法は、価格が上昇した株式（過大評価されている可能性がある）を過大に組み入れ、価格が下落した株式（過小評価されている可能性がある）を過小に組み入れることにつながる。この効果はモメンタム投資戦略（momentum investment strategy）に類似しており、時間の経過とともに価格が最も上昇した証券が指数内で最大のウェイトを持つようになる。

ファンダメンタル加重（Fundamental Weighting）

ファンダメンタル加重（fundamental weighting）は、各構成銘柄のウェイトを決定する際に、株価とは独立した企業規模の指標を用いることで、時価総額加重の欠点に対処しようとするものである。これらの指標には、簿価（book value）、キャッシュフロー（cash flow）、売上高（revenues）、利益（earnings）、配当（dividends）、従業員数（number of employees）などが含まれる。

一部のファンダメンタル指数は、総配当（total dividends）のような単一の指標を用いて構成銘柄を加重する一方で、複数の指標のウェイトを組み合わせて合成値（composite value）を作成し、それを加重に用いるものもある。

ファンダメンタル加重の最も重要な特性は、「バリュー（value）」傾向を持つ指数をもたらす点である。すなわち、ファンダメンタル加重指数は、簿価、利益、配当などの市場価値に対する比率が、時価総額加重指数よりも高くなる。また、時価総額加重指数のモメンタム的な「効果（effect）」とは対照的に、ファンダメンタル加重指数は一般に逆張り（contrarian）的な「効果」を持つ。すなわち、ポートフォリオがリバランスされる際に、相対的価値が上昇した証券から離れ、相対的価値が下落した証券へとウェイトがシフトする。

2.04 | Index Management: Rebalancing and Reconstitution

モジュール2：証券市場指数（Security Market Indexes） 2.04｜指数管理：リバランシングとリコンスティテューション（Index Management: Rebalancing and Reconstitution）

 Learning Outcome

The candidate should be able to:

  • 指数のリバランシング（rebalancing）およびリコンスティテューション（reconstitution）を説明する

これまで指数の構築について説明してきた。指数管理（index management）は、残りの2つの問いを含む：

  • 指数はいつリバランスされるべきか？
  • 証券の選択およびウェイトの決定はいつ再検討されるべきか？

リバランシング（Rebalancing）

リバランシング（rebalancing）とは、指数に含まれる構成銘柄（constituent securities）のウェイトを調整することを指す。各証券のウェイトを指数の加重方法（weighting method）と一致させるために、指数提供者（index provider）は、定期的なスケジュール（リバランス日、rebalancing dates）に基づいてウェイトを調整する。通常は四半期ごと（quarterly）に行われる。構成銘柄のウェイトは市場価格の変動に伴って変化するため、リバランシングが必要となる。例えば、等加重指数（equal-weighted index）（図表3（Exhibit 3））において、期末時点では各証券のウェイトはもはや等しく（20％）なくなっている：

証券A（Security A） 19.93％

指数をリバランスする際には、最も高いリターンを示した証券DおよびEのウェイトを引き下げ、最も低いリターンを示した証券A、B、およびCのウェイトを引き上げることになる。このように、リバランシングは指数内に売買回転（turnover）を生じさせる。

価格加重指数（price-weighted indexes）は、各構成銘柄のウェイトがその価格によって決定されるため、リバランスされない。時価総額加重指数（market-capitalization-weighted indexes）については、指数自体が概ね自動的にリバランスされるため、リバランシングはそれほど重要ではない。例えば、当該時価総額加重指数では、証券Cのウェイトは市場価格が36％下落したことを反映して、自動的に10.96％から6.91％へ低下した。時価総額ウェイトは、リバランス日間においては、合併（mergers）、買収（acquisitions）、清算（liquidations）、およびその他のコーポレートアクション（corporate actions）を反映するためにのみ調整される。

リコンスティテューション（Reconstitution）

リコンスティテューション（reconstitution）とは、指数の構成銘柄を変更するプロセスである。これは、ポートフォリオマネージャーが自身のポートフォリオの銘柄を変更することに類似している。リコンスティテューションはリバランシングサイクルの一部である。リコンスティテューション日（reconstitution date）とは、指数提供者が構成銘柄を見直し、指数への組入基準（criteria）を再適用し、維持・除外・追加する証券を選定する日である。基準を満たさなくなった構成銘柄は、基準を満たす証券に置き換えられる。改訂された構成銘柄リストが決定された後、加重方法が再適用される。指数は、ターゲット市場の変化（破綻（bankruptcies）、上場廃止（de-listings）、合併（mergers）、買収（acquisitions）など）を反映するため、および／または選定委員会（selection committee）の判断を反映するためにリコンスティテューションされる。

リコンスティテューションは、特に時価総額加重指数において、さまざまな形で売買回転（turnover）を生じさせる。ある証券が除外され、別の証券が追加される場合、指数提供者は指数の時価総額加重を維持するために他の証券のウェイトも調整しなければならない。

リコンスティテューションの頻度は、広く利用されている指数およびその構成銘柄にとって重要な問題である。例えば、ラッセル2000指数（Russell 2000 Index）は年次でリコンスティテューションを行う。この指数は多くの投資ファンドのベンチマーク（benchmark）として使用されており、毎年リコンスティテューションの前には、これらのファンドの運用者が指数に追加されると予想する銘柄を購入し（その価格を押し上げ）、除外されると予想する銘柄を売却する（その価格を押し下げる）。図表6（Exhibit 6）は、これらの意思決定がもたらす潜在的な影響の歴史的事例を示している。2009年4月下旬から、一部の運用者はウラニウム・エナジー（Uranium Energy Corporation, UEC）が再構成後のラッセル2000指数に組み入れられると考え、その株式を取得し価格を押し上げ始めた。6月12日、ラッセル（Russell）はUECをラッセル2000指数およびラッセル3000指数の暫定的な追加銘柄として公表した。² その時点までに株価は300％以上上昇していた。発表後の数週間も投資家は株価をさらに押し上げ、6月30日のリコンスティテューション日において株価は2.90米ドル（USD2.90）で引け、四半期でほぼ400％の上昇となった。

2.05 | Uses of Market Indexes

モジュール2：証券市場指数（Security Market Indexes） 2.05｜市場指数の用途（Uses of Market Indexes）

Learning Outcome

The candidate should be able to:

  • 証券市場指数（security market indexes）の用途を説明する

指数は当初、特定の証券市場（security market）がある日にどのようなパフォーマンスを示したかを把握するために作られた。現代の金融理論（modern financial theory）の発展に伴い、投資管理（investment management）におけるその用途は大きく拡大した。指数の主な用途には以下が含まれる：

  • 市場センチメント（market sentiment）の指標；
  • リターン、システマティック・リスク（systematic risk）、およびリスク調整後パフォーマンス（risk-adjusted performance）を測定およびモデル化するための代理変数（proxies）；
  • アセットアロケーションモデル（asset allocation models）における資産クラス（asset classes）の代理変数（proxies）；
  • アクティブ運用ポートフォリオ（actively managed portfolios）のベンチマーク（benchmarks）；および
  • インデックスファンド（index funds）や上場投資信託（exchange-traded funds, ETFs）などの投資商品のモデルポートフォリオ（model portfolios）。

証券市場指数（security market indexes）を利用する投資家は、自身のニーズに最も適した指数を選択するために、さまざまな指数がどのように構築されているかを理解しておく必要がある。

市場センチメントの指標（Gauges of Market Sentiment）

株式市場指数（stock market indexes）の本来の目的は、投資家の信頼感（investor confidence）または市場センチメント（market sentiment）の指標を提供することであった。市場参加者の集合的な意見を示す指標として、指数は投資家の態度や行動を反映する。ダウ・ジョーンズ工業平均（Dow Jones Industrial Average）は長い歴史を持ち、メディアで頻繁に引用され、依然として人気のある市場センチメントの指標である。しかし、この指数は毎日取引される数千の米国株のうちわずか30銘柄で構成されているため、投資家全体の態度や「市場」全体を正確に反映しているとは限らない。

リターン、システマティック・リスク、およびリスク調整後パフォーマンスの測定およびモデル化のための代理変数（Proxies for Measuring and Modeling Returns, Systematic Risk, and Risk-Adjusted Performance）

資本資産価格モデル（capital asset pricing model, CAPM）は、ベータ（beta）を市場全体に対する証券のシステマティック・リスクとして定義する。CAPMにおける市場ポートフォリオ（market portfolio）はすべてのリスク資産で構成される。市場ポートフォリオのパフォーマンスを表すために、投資家は広範な指数を用いる。例えば、東証株価指数（Tokyo Price Index, TOPIX）やS&P 500は、それぞれ日本および米国における市場ポートフォリオの代理として機能し、システマティック・リスクおよび市場リターンの測定やモデル化に用いられる。

証券市場指数は、リスク調整後パフォーマンスを測定する際の市場の代理としても機能する。アクティブ運用ポートフォリオのベータ（beta）は、投資家が同じシステマティック・リスク水準を持つパッシブな代替ポートフォリオを構築することを可能にする。例えば、グローバル株式のアクティブ運用ポートフォリオのベータがMSCI World Indexに対して0.95である場合、投資家はポートフォリオの95％をMSCI World Indexファンドに投資し、残り5％を現金（cash）で保有することで、同じシステマティック・リスクを持つパッシブポートフォリオを構築できる。アルファ（alpha）は、アクティブ運用ポートフォリオのリターンとパッシブポートフォリオのリターンの差であり、リスク調整後リターンまたは投資パフォーマンスの指標である。アルファは、運用者の能力（またはその欠如）、取引コスト（transaction costs）、および手数料（fees）に起因する可能性がある。

アセットアロケーションモデルにおける資産クラスの代理変数（Proxies for Asset Classes in Asset Allocation Models）

指数は特定の証券グループのリスクおよびリターン特性を示すため、アセットアロケーションモデルにおいて資産クラスの代理変数として重要な役割を果たす。指数は、異なる資産クラスのリスクおよびリターンをモデル化するために使用される過去データを提供する。

アクティブ運用ポートフォリオのベンチマーク（Benchmarks for Actively Managed Portfolios）

投資家は、アクティブ運用ポートフォリオマネージャーのパフォーマンスを評価するために、しばしば指数をベンチマークとして使用する。ベンチマークとして選択される指数は、運用者が採用する投資戦略を反映している必要がある。例えば、グローバルな小型株（small-capitalization stocks）に投資するアクティブ運用者は、FTSEグローバル小型株指数（FTSE Global Small Cap Index）のようなベンチマークで評価されるべきである。この指数は2018年8月時点で、47か国にわたる約4,400の流動性の高い小型株で構成されている。

ベンチマークとして用いる指数の選択は重要である。不適切な指数を用いると、アクティブ運用者の投資パフォーマンスに関して誤った結論を導く可能性がある。例えば、小型株運用者が小型株指数には劣後したが、広範な株式市場指数には勝っていたとする。投資家が広範な市場指数をベンチマークとして用いた場合、その運用者は手数料に見合う成果を上げており、継続または追加資産の運用を任せるべきであると結論づける可能性がある。一方、小型株指数をベンチマークとして用いると、全く異なる結論に至る可能性がある。

投資商品のモデルポートフォリオ（Model Portfolios for Investment Products）

指数はまた、新たな投資商品の開発の基盤としても機能する。アクティブ運用ポートフォリオのベンチマークとして指数を用いることにより、一部の投資家は、ベンチマークそのものに投資すべきであると考えるようになった。CAPMの結論である「投資家は市場ポートフォリオを保有すべきである」という考えに基づき、広範な市場指数ファンドが市場ポートフォリオの代理として開発された。

投資運用会社（investment management firms）は当初、機関投資家（institutional investors）向けに指数ポートフォリオを開発・運用していた。その後、投資信託会社（mutual fund companies）は個人投資家向けにインデックスファンドを提供し始めた。さらに、投資運用会社は上場投資信託（exchange-traded funds, ETFs）を導入した。これらはインデックス投資信託と同様に運用されるが、株式のように取引される。

最初のETFは既存の指数に基づいていた。ETFの人気が高まるにつれて、指数提供者はETFを組成する特定の目的のために新たな指数を開発し、その結果、対応するETFを持つ多数の狭く定義された指数が誕生した。例えば、VanEck Vectors Vietnam ETFは、投資家がベトナムの株式市場に投資することを可能にする。

投資家のニーズに応じた指数の選択肢は非常に広範である。指数提供者は、投資家のニーズに応える新たな指数の開発機会を常に模索している。

2.06 | Equity indexes

モジュール2：証券市場指数（Security Market Indexes） 2.06｜株式指数（Equity Indexes）

 Learning Outcome

The candidate should be able to:

  • 株式指数（equity indexes）の種類を説明する
  • 証券市場指数（security market indexes）の種類を比較する

株式指数には、広範市場指数（broad market）、マルチマーケット指数（multi-market）、セクター指数（sector）、スタイル指数（style）など、多様な種類が存在する。

広範市場指数（Broad Market Indexes）

広範株式市場指数（broad equity market index）は、その名のとおり、特定の株式市場全体を表し、通常は選定された市場の90％以上を代表する証券を含む。例えば、上海総合指数（Shanghai Stock Exchange Composite Index, SSE）は、上海証券取引所（Shanghai Stock Exchange）で取引されるすべての株式を対象とした時価総額加重指数（market-capitalization-weighted index）である。米国では、ウィルシャー5000トータルマーケット指数（Wilshire 5000 Total Market Index）が、価格データが入手可能なすべての米国株式を含む時価総額加重指数であり、米国株式市場全体を表すことを目的としている。³ また、時価総額上位3,000銘柄で構成されるラッセル3000（Russell 3000）は、米国株式市場の約98％を代表している。

マルチマーケット指数（Multi-Market Indexes）

マルチマーケット指数（multi-market indexes）は、通常、異なる国や地域の指数で構成され、複数の証券市場を表すよう設計されている。これらの指数は、複数の国別市場、地理的地域、経済発展段階別グループ、場合によっては世界全体を表すことがある。世界指数（world indexes）は、特定の国や地域に偏らず、グローバルな株式投資を行う投資家にとって重要である。多くの指数提供者は、マルチマーケット株式指数のファミリーを提供している。

マルチマーケット指数におけるファンダメンタル加重（Fundamental Weighting in Multi-Market Indexes）

一部の指数提供者は、各国・地域内の証券を時価総額（market capitalization）で加重し、その後、各国・地域をその相対的な国内総生産（GDP）に比例して指数全体で加重することで、マルチマーケット指数におけるファンダメンタル加重を実現している。GDP加重指数（GDP-weighted indexes）は、最初に作成されたファンダメンタル加重指数の一部である。1987年にMSCIによって導入され、当時の時価総額加重MSCI EAFE指数（MSCI EAFE Index）において日本株が60％のウェイトを占めていた問題に対処するために設計された。GDP加重指数は、日本株への配分を半分に削減した。⁴

セクター指数（Sector Indexes）

セクター指数（sector indexes）は、消費財（consumer goods）、エネルギー（energy）、金融（finance）、ヘルスケア（health care）、テクノロジー（technology）などの異なる経済セクターを、国別、地域別、またはグローバルベースで表し、追跡する。経済の各セクターは景気循環（business cycle）の中で異なる動きを示すため、一部の投資家は特定のセクターへのエクスポージャーを過大配分（overweight）または過小配分（underweight）することを志向する場合がある。

セクター指数はファミリーとして構成され、ファミリー内の各指数が1つの経済セクターを表す。通常、セクター指数ファミリーを集約すると広範市場指数に相当する。経済セクター分類は、グローバル、地域別、または国別で適用できるが、普遍的に合意された分類方法は存在しない。

セクター指数はパフォーマンス分析（performance analysis）において重要な役割を果たす。なぜなら、ポートフォリオマネージャーが銘柄選択（stock selection）に優れているのか、セクター配分（sector allocation）に優れているのかを判断する手段を提供するためである。また、セクター指数はセクター特化型ETF（sector-specific ETFs）やその他の投資商品のモデルポートフォリオとしても機能する。

スタイル指数（Style Indexes）

スタイル指数（style indexes）は、時価総額（market capitalization）、バリュー（value）、グロース（growth）、またはこれらの組み合わせに基づいて分類された証券群を表す。これらは、グロース投資家（growth investor）、バリュー投資家（value investor）、小型株投資家（small-cap investor）など、特定の投資スタイルを反映することを目的としている。

時価総額（Market Capitalization）

時価総額指数（market-capitalization indexes）は、大型株（large cap）、中型株（midcap）、小型株（small cap）という主要な時価総額カテゴリーに基づいて証券を分類する。これらのカテゴリーに関する統一的な定義は存在しないため、大型株と中型株、中型株と小型株の区別や、小型株指数に含まれるための最小時価総額は指数ごとに異なる。分類は絶対的な時価総額（例：1億ユーロ未満）または相対的な時価総額（例：最小の2,500銘柄）に基づく場合がある。

バリュー／グロース分類（Value/Growth Classification）

一部の指数は、株式をバリュー株（value stocks）またはグロース株（growth stocks）として分類することに基づいて構成される。異なる指数提供者は、バリュー株とグロース株を区別するために、異なる要因や評価指標（valuation ratios）（低い株価純資産倍率（price-to-book ratios）、低い株価収益率（price-to-earnings ratios）、高い配当利回り（dividend yields）など）を使用する。

時価総額およびバリュー／グロース分類（Market Capitalization and Value/Growth Classification）

3つの時価総額グループとバリュー／グロース分類を組み合わせると、基本的な6つのスタイル指数カテゴリーが得られる：

大型株バリュー（Large-Cap Value） 大型株グロース（Large-Cap Growth）
中型株バリュー（Mid-Cap Value） 中型株グロース（Mid-Cap Growth）
小型株バリュー（Small-Cap Value） 小型株グロース（Small-Cap Growth）

指数は異なる規模および評価基準を用いるため、小型株バリュー（small-cap value）など特定のスタイルを表すよう設計された指数の構成銘柄は、時に大きく異なることがある。

評価指標（valuation ratios）や時価総額は時間とともに変化するため、銘柄はリコンスティテューション日（reconstitution dates）において、あるスタイル指数カテゴリーから別のカテゴリーへ頻繁に移行する。その結果、スタイル指数は一般に広範市場指数よりもはるかに高い売買回転（turnover）を持つ。

2.07 | Fixed-income indexes

モジュール2：証券市場指数（Security Market Indexes） 2.07｜債券指数（Fixed-Income Indexes）

 Learning Outcome

The candidate should be able to:

  • 債券指数（fixed-income indexes）の種類を説明する
  • 証券市場指数（security market indexes）の種類を比較する

多様な債券指数（fixed-income indexes）が存在するが、債券市場（fixed-income markets）および債券証券（fixed-income securities）の特性により、債券指数の構築およびレプリケーション（replication）にはいくつかの重要な課題がある。これらの課題には、債券ユニバース（fixed-income universe）における証券数、価格データの入手可能性（availability of pricing data）、および証券の流動性（liquidity）が含まれる。

構築（Construction）

債券ユニバースには、政府（governments）、政府機関（government agencies）、および企業（corporations）が発行する証券が含まれる。これらの発行体は、それぞれ異なる特性を持つ多様な債券を発行する可能性がある。その結果、債券証券の数は株式証券の数よりもはるかに多い。特定の債券市場またはセグメントを表すために、指数には数千の異なる証券が含まれることがある。時間の経過とともに、これらの債券は満期（mature）を迎え、発行体は資金調達ニーズを満たすために新たな証券を発行するため、債券指数には売買回転（turnover）が生じる。

指数構築におけるもう一つの課題は、債券市場が主にディーラーマーケット（dealer markets）である点である。これは、企業（ディーラー、dealers）が特定の証券に割り当てられ、自らの在庫（inventory）を用いて売買を行うことで、その証券の流動性のある市場を形成する責任を負うことを意味する。さらに、多くの証券は頻繁に取引されないため、比較的非流動的（illiquid）である。その結果、指数提供者（index providers）は指数を更新するために構成銘柄の現在価格を得るためにディーラーに連絡する必要があるか、または類似の特性を持つ取引された債券の価格を用いて構成銘柄の価格を推定しなければならない。

これらの課題により、同じ市場を表す指数であっても、含まれる債券数が異なる場合がある。債券証券の数が非常に多いこと、および一部の証券の流動性が低いことにより、投資家にとって債券指数を再現し、そのパフォーマンスを複製することは、株式指数と比較してよりコストが高く、困難となっている。

債券指数の種類（Types of Fixed-Income Indexes）

ゼロクーポン債（zero-coupon bonds）からオプション内蔵債（embedded options）（すなわちコーラブル債（callable bonds）やプッタブル債（putable bonds））まで、多様な債券証券が存在するため、さまざまな種類の債券指数が存在する。株式と同様に、債券証券は発行体の経済セクター（economic sector）、地理的地域（geographic region）、またはその地域の経済発展段階（economic development）に基づいて分類される。また、債券証券は以下の次元でも分類される：

  • 発行体の種類（type of issuer）（政府、政府機関、企業）
  • 資金調達の種類（type of financing）（一般財源債（general obligation）、担保付（collateralized））
  • 支払い通貨（currency of payments）
  • 満期（maturity）
  • 信用の質（credit quality）（投資適格（investment grade）、ハイイールド（high yield）、信用格付け（credit agency ratings））；および
  • インフレ保護の有無（absence or presence of inflation protection）

債券指数はこれらのさまざまな次元に基づいて構築され、以下のように分類される：

  • 総合または広範市場指数（aggregate or broad market indexes）
  • 市場セクター指数（market sector indexes）
  • スタイル指数（style indexes）
  • 経済セクター指数（economic sector indexes）；および
  • ハイイールド（high-yield）、インフレ連動（inflation-linked）、新興市場（emerging market）などの特化型指数（specialized indexes）

最初に作成された債券指数であるブルームバーグ・バークレイズ米国総合債券指数（Bloomberg Barclays US Aggregate Bond Index）（旧バークレイズ・キャピタル総合債券指数（Barclays Capital Aggregate Bond Index））は、単一国の総合指数の一例である。この指数は米国の債券市場全体を表すよう設計されており、米国国債（US Treasury）、政府関連債（government-related）、社債（corporate）、住宅ローン担保証券（mortgage-backed）、資産担保証券（asset-backed）、商業用住宅ローン担保証券（commercial mortgage-backed）を含む約8,000の証券で構成されている。

総合指数は、市場セクター（政府、政府機関、担保付、企業）、スタイル（満期、信用の質）、経済セクター、またはその他の特性によってさらに細分化され、より限定的な指数を作成することができる。指数でよく見られる区分の一つに、投資適格（investment grade）（例：Standard & Poor’sの信用格付けでBBB−以上）とハイイールド証券の区別がある。投資適格指数は通常、満期（短期、中期、長期）および信用格付け（AAA、BBBなど）によってさらに細分化される。⁵ 債券指数の多様性は、さまざまな次元に基づいて債券証券が区分されていることを反映している。

2.08 | Indexes for Alternative Investments

モジュール2：証券市場指数（Security Market Indexes） 2.08｜オルタナティブ投資の指数（Indexes for Alternative Investments）

 Learning Outcome

The candidate should be able to:

  • オルタナティブ投資（alternative investments）を表す指数を説明する
  • 証券市場指数（security market indexes）の種類を比較する

多くの投資家は、株式（equities）や債券（fixed income）以外の資産クラス（asset classes）に投資することで、ポートフォリオのリスクを低減したりパフォーマンスを向上させたりしようとする。オルタナティブ資産（alternative assets）や投資戦略への関心の高まりにより、オルタナティブ投資の広範なクラスを表すよう設計された指数が作られている。最も広く追跡されているオルタナティブ投資クラスの3つは、コモディティ（commodities）、不動産（real estate）、ヘッジファンド（hedge funds）である。

コモディティ指数（Commodity Indexes）

コモディティ指数（commodity indexes）は、1つまたは複数のコモディティ（commodities）に関する先物契約（futures contracts）で構成される。これには、農産物（agricultural products）（米（rice）、小麦（wheat）、砂糖（sugar））、畜産物（livestock）（牛（cattle）、豚（hogs））、貴金属および一般金属（precious and common metals）（金（gold）、銀（silver）、銅（copper））、エネルギー商品（energy commodities）（原油（crude oil）、天然ガス（natural gas））などが含まれる。

一部のコモディティ指数は同じコモディティを含む場合があるが、加重方法（weighting method）が異なるため、リターンは異なる場合がある。コモディティ指数には時価総額のような明確な加重基準が存在しないため、指数提供者は独自の加重方法を考案する。例えば、トムソン・ロイター／コア・コモディティCRB指数（Thomson Reuters/Core Commodity CRB Index, TR/CC CRB Index）（旧コモディティ・リサーチ・ビューロー指数（Commodity Research Bureau, CRB Index））のような指数は、一定数のコモディティを等加重（equally）で組み入れている。一方、S&P GSCIは流動性指標（liquidity measures）と世界生産量（world production values）を組み合わせた加重方式を採用し、価格が上昇したコモディティにより高いウェイトを配分する。また、委員会によって固定ウェイトが決定される指数も存在する。

加重方法の違いは、特定のコモディティへのエクスポージャーにも大きな差をもたらす可能性がある。例えば、2018年のS&P GSCIは、エネルギーセクターのウェイトがCRB指数より約50％高く、農業セクターのウェイトが約40％低い。これらの違いにより、リスクおよびリターン特性が大きく異なる指数が生じる。コモディティ指数とは異なり、同一市場を対象とする広範な株式指数や債券指数は、類似したリスクおよびリターン特性を持つ。

コモディティ指数のパフォーマンスは、実際のコモディティとは大きく異なる場合がある。これは、指数が現物ではなく先物契約で構成されているためである。指数リターンは、原資産の価格変動以外の要因にも影響される。なぜなら、先物契約は継続的にロールオーバー（roll over）（すなわち満期が近い契約を新たな契約に置き換える）される必要があるためである。コモディティ指数のリターンは、無リスク金利（risk-free interest rate）、先物価格の変化、およびロールイールド（roll yield）を反映する。そのため、コモディティ指数のリターンは、原資産価格の変動に基づくリターンと大きく異なる可能性がある。

不動産投資信託指数（Real Estate Investment Trust Indexes）

不動産指数（real estate indexes）は、不動産証券市場だけでなく、不動産市場そのものも表す。不動産市場は、取引頻度が低く価格情報も限られる非常に非流動的（illiquid）な市場および資産クラスである。不動産指数は、鑑定指数（appraisal indexes）、リピートセールス指数（repeat sales indexes）、および不動産投資信託指数（real estate investment trust, REIT indexes）に分類される。

REIT指数（REIT indexes）は、上場REIT（publicly traded REITs）の株式で構成される。REIT（REITs）は、不動産の所有またはモーゲージへの投資を通じて不動産に投資することを目的として設立された公開または非公開の法人である。上場REITの株式は世界各地の証券取引所で取引され、商業用不動産への投資手段として人気がある。REIT指数は継続的な市場価格を持つ上場REITに基づいているため、その指数値は継続的に計算される。

図表11（Exhibit 11）に示されるFTSE EPRA/NAREITグローバルREIT指数ファミリー（FTSE EPRA/NAREIT global family of REIT indexes）は、世界の不動産株の動向を表すことを目的としており、欧州上場不動産協会（European Public Real Estate Association, EPRA）および全米不動産投資信託協会（National Association of Real Estate Investment Trusts, NAREIT）の協力を得て構成されている。

ヘッジファンド指数（Hedge Fund Indexes）

ヘッジファンド指数（hedge fund indexes）は、ヘッジファンドのリターンを反映する。ヘッジファンドは通常、レバレッジ（leverage）やロング・ショート戦略（long and short investment strategies）を用いるプライベート投資ビークルである。

多くの調査機関（research organizations）がヘッジファンドのリターンデータベースを維持し、それらのリターンを指数として集約している。これらのデータベース指数は、ヘッジファンド全体または戦略別に、グローバルレベルでのパフォーマンスを表すことを目的としている。これらの指数の多くは等加重（equal weighted）であり、特定のデータベースに含まれるヘッジファンドのパフォーマンスを表している。

多くの調査機関は、ヘッジファンドの自発的な協力に依存してパフォーマンスデータを収集している。しかし、ヘッジファンドは非規制（unregulated）の存在であり、投資家以外の第三者にパフォーマンスを報告する義務はない。そのため、各ヘッジファンドはどのデータベースに報告するかを自ら決定する。この結果、指数提供者が構成銘柄を決定するのではなく、構成銘柄が指数を決定することになる。

多くの場合、ヘッジファンドは1つのデータベースのみにパフォーマンスを報告する。その結果、異なる指数間で対象ファンドの重複はほとんどない。構成銘柄の重複が少ないため、異なるグローバルヘッジファンド指数は、同一期間においてヘッジファンド業界のパフォーマンスを大きく異なる形で示すことがある。

自発的なパフォーマンス報告のもう一つの結果として、生存バイアス（survivorship bias）の可能性があり、パフォーマンスの表現が不正確になる可能性がある。これは、パフォーマンスが低いヘッジファンドがデータベースへの報告を控えたり、報告を停止したりする可能性があり、その結果、それらのリターンが指数の計測から除外されることを意味する。その結果、指数は実際のヘッジファンドのパフォーマンスではなく、良好なパフォーマンスを示しているファンドのパフォーマンスを反映する傾向がある。

2.09 | Summary

モジュール2：証券市場指数（Security Market Indexes）2.09｜要約（Summary）

本リーディングでは、証券市場指数（security market indexes）の構築、管理、および利用について説明し、例示する。また、さまざまな種類の指数についても論じる。証券市場指数は投資家にとって非常に有用なツールであり、多様な証券市場、市場セグメント、および資産クラス（asset classes）を代表する数千の指数の中から選択することができる。これらの指数は、主要な資産クラスのグローバル市場を表すものから、特定の地理的市場におけるオルタナティブ投資（alternative investments）を表すものまで多岐にわたる。証券市場指数を有効に活用するためには、投資家はその構築方法を理解し、選択した指数が自身の目的に適しているかを判断しなければならない。しばしば、ある目的に適した指数が他の目的には適さない場合がある。指数の利用者は、自身のニーズに最も適した指数を選択するために、さまざまな指数がどのように構築されているかを理解している必要がある。

本リーディングで示された主なポイントは以下のとおりである。

  • 証券市場指数は、異なる対象市場（証券市場、市場セグメント、または資産クラス）の価値を測定することを目的としている。
  • 証券市場指数に組み入れられる構成証券（constituent securities）は、対象市場を代表することを目的として選定される。
  • プライスリターン指数（price return index）は、構成証券の価格のみを反映する。
  • トータルリターン指数（total return index）は、構成証券の価格だけでなく、指数の開始以降に受け取ったすべての収益の再投資も反映する。
  • 指数の構成銘柄に対するウェイト付け方法は、価格加重（price weighting）や等ウェイト（equal weighting）といった非常に単純なものから、時価総額加重（market-capitalization weighting）やファンダメンタル加重（fundamental weighting）といったより複雑なものまで幅広い。
  • 指数構築における選択、特にウェイト付け方法の選択は、指数の評価（valuation）およびリターンに影響を与える。
  • 指数の管理には、(1) 適切なウェイトを維持するための定期的なリバランス（rebalancing）と、(2) 指数が望ましい対象市場を代表することを確保するためのリコンスティテューション（reconstitution）が含まれる。
  • リバランスとリコンスティテューションは、指数における売買回転（turnover）を生み出す。リコンスティテューションは、現在および将来の構成銘柄の価格に大きな影響を与える可能性がある。
  • 指数はさまざまな目的に利用される。市場センチメント（market sentiment）を測定し、アクティブ運用ポートフォリオ（actively managed portfolios）のベンチマーク（benchmarks）として機能する。また、システマティックリスク（systematic risk）やリスク調整後パフォーマンス（risk-adjusted performance）を測定するための代理指標（proxies）として機能する。さらに、資産配分モデル（asset allocation models）における資産クラスの代理や、投資商品のモデルポートフォリオとしても用いられる。
  • 投資家は、株式（equity）、債券（fixed-income）、コモディティ（commodity）、不動産（real estate）、およびヘッジファンド指数（hedge fund indexes）など、さまざまな資産クラスを代表する証券市場指数から選択することができる。
  • ほとんどの資産クラスにおいて、指数提供者（index providers）は、広範な市場指数から、発行体の地理的地域、経済発展グループ、経済セクター、またはその他の要因に基づく高度に特化した指数まで、幅広い種類の指数を提供している。
  • 証券市場指数の適切な利用は、その構築および管理の理解に依存する。

Glossary: Security Market Indexes

モジュール2：証券市場指数（Security Market Indexes）用語集（Glossary）

 

C

Constituent securities

指数に関して、指数を構成する個々の証券（individual securities）。

D

Divisor

プライスリターン指数（price return index）の値を算出するために用いられる数値（分母）。これは指数の開始時（inception）に初期設定され、その後、構成証券の価格変化とは無関係な指数値の変動を避けるため、必要に応じて指数提供者（index provider）によって調整される。

E

Equal weighting

各構成証券に対して、開始時に等しいウェイトを割り当てる指数のウェイト付け方法（equal weighting）。

F

Float-adjusted market-capitalization weighting

各構成証券に割り当てられるウェイトが、その時価総額（market capitalization）を浮動株（market float）で調整することによって決定される指数のウェイト付け方法（float-adjusted weighting）。

Fundamental weighting

各構成証券に割り当てられるウェイトが、その企業の規模（size）に基づいて決定される指数のウェイト付け方法。これは、構成証券の価格に依存しない指標を用いることで、時価総額加重（market-capitalization weighting）の欠点に対処しようとするものである（fundamental weighting）。

H

Hedge funds

公開株式（public equities）や上場固定収益資産（publicly traded fixed-income assets）、プライベートキャピタル（private capital）、および／または実物資産（real assets）に投資する可能性のあるプライベート投資ビークル（private investment vehicles）であるが、投資対象そのものではなく投資アプローチによって区別される。

M

Market-capitalization weighting

各構成証券に割り当てられるウェイトが、その時価総額を指数内のすべての証券の総時価総額で割ることによって決定される指数のウェイト付け方法。バリュー加重（value weighting）とも呼ばれる。

Market float

投資家が取引可能な株式数（number of shares available to the investing public）。

Multi-market indexes

異なる国の指数から構成され、複数の証券市場を表すことを目的として設計された指数。

P

Price index

財・サービスのバスケット（basket of goods and services）の平均価格を表す。

Price return

指数またはポートフォリオに含まれる証券の価格上昇（price appreciation）または価格変化率のみを測定する。

Price return index

構成証券の価格上昇または価格変化率のみを反映する指数。プライス指数（price index）とも呼ばれる。

Price weighting

各構成証券に割り当てられるウェイトが、その価格をすべての構成証券の価格の合計で割ることによって決定される指数のウェイト付け方法（price weighting）。

R

Rebalancing

指数における構成証券のウェイトを調整すること（rebalancing）。

S

Sector indexes

消費財（consumer goods）、エネルギー（energy）、金融（finance）、ヘルスケア（health care）、テクノロジー（technology）などの異なる経済セクターを、国別、地域別、またはグローバルベースで表し追跡する指数。

Security market index

特定の証券市場、市場セグメント、または資産クラスを代表する証券ポートフォリオ（portfolio of securities）。

T

Total return

指数またはポートフォリオに含まれる証券の価格上昇または価格変化率に加え、期間中に受け取った収益も測定する。

Total return index

構成証券の価格上昇または価格変化率に加え、開始時以降に受け取ったすべての収益を反映する指数（total return index）。

Real World Applications: Bloomberg's Index Browser

モジュール2：証券市場指数（Security Market Indexes）実務応用（Real World Applications）：ブルームバーグのインデックス・ブラウザ（Bloomberg's Index Browser）

ブルームバーグ（Bloomberg）は、金融プロフェッショナルが業務全体を通じて世界的に利用している主要な市場データプラットフォームの一つである。本オプションセクションでは、CFAプログラム（CFA Program）のカリキュラムの実務応用をブルームバーグ上で探る。

{IN} ブルームバーグ・インデックス・ブラウザ（BLOOMBERG INDEX BROWSER）

ブルームバーグのIN機能（IN function）を使用すると、資産クラス（asset class）別に最も広く利用されている指数を閲覧することができる。

 

「IN <GO> は、ブルームバーグのグローバルなマルチ資産クラス（multi-asset class）の指数ファミリー（family of indices）（ブルームバーグ債券指数（Bloomberg Fixed Income indices）を含む）の最新のパフォーマンスデータおよび履歴を表示できる機能であり、リスク分析（risk analysis）に基づいて情報に基づく投資判断を行うために、期間収益（periodic returns）や統計データ（statistical data）を分析することができる。」—ブルームバーグ

 

この機能を利用するには、ブルームバーグのコマンドラインに「IN」と入力し、Enterまたは<GO>を押す。タブ（tabs）を使用して分析したいブルームバーグ指数の資産クラスを選択する。サブタブ（subtabs）では、各資産クラスが関連する指数ファミリーごとに分類されている。
